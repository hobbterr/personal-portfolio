# 本地检查和上线维护流程

这份文档用于以后每次修改网站后，检查本地效果并更新线上 Vercel 版本。

## 1. 本地预览检查

用于查看网页实际效果。

在项目根目录运行：

```bash
npm.cmd run dev
```

打开浏览器访问：

```txt
http://localhost:5173/
```

主要检查：

- 首页是否正常显示
- 首屏视频是否正常
- 菜单是否可以点击
- 品牌视觉、活动视觉分类页是否正常
- 视频弹窗是否正常播放
- 瀑布流是否正常排列
- 移动端效果是否正常

`dev` 是给你本地预览用的。

## 2. 手机局域网预览

如果想用手机查看移动端真实效果，不要只运行：

```bash
npm.cmd run dev
```

因为普通 `dev` 默认主要给电脑本机访问。

手机预览请运行：

```bash
npm.cmd run dev -- --host 0.0.0.0
```

然后查看电脑的局域网 IP。

在 PowerShell 里运行：

```bash
ipconfig
```

找到类似：

```txt
IPv4 地址 . . . . . . . . . . . . : 192.168.1.23
```

手机和电脑需要连接同一个 Wi-Fi。

然后在手机浏览器打开：

```txt
http://电脑IPv4地址:5173/
```

例如：

```txt
http://192.168.1.23:5173/
```

注意：

- 地址前面是 `http://`，不是 `https://`
- 手机和电脑必须在同一个 Wi-Fi
- PowerShell 里的 Vite 服务窗口不能关闭
- 如果打不开，可能是 Windows 防火墙拦截了 Node.js，需要允许访问专用网络

## 3. 上线前构建检查

用于确认项目能不能成功部署到 Vercel。

在项目根目录运行：

```bash
npm.cmd run build
```

如果看到类似：

```txt
✓ built in ...
```

说明构建成功，通常 Vercel 也可以正常部署。

`build` 是上线前检查用的。

## 4. 提交本地修改

确认本地预览和构建都没问题后，提交修改：

```bash
git add .
git commit -m "update website"
```

如果你想写更具体的提交说明，也可以改成：

```bash
git commit -m "update portfolio works"
```

## 5. 推送到 GitHub

提交完成后运行：

```bash
git push
```

推送成功后，GitHub 仓库会更新。

## 6. Vercel 自动部署

GitHub 更新后，Vercel 会自动开始重新部署。

你可以打开 Vercel 项目页面查看部署状态。

部署完成后，刷新线上网址即可看到新版网站。

## 7. 推荐完整流程

每次修改网站后，推荐按这个顺序：

```bash
npm.cmd run dev
npm.cmd run build
git add .
git commit -m "update website"
git push
```

说明：

- `npm.cmd run dev`：本地看效果
- `npm.cmd run build`：确认能上线
- `git add .`：把修改加入 Git
- `git commit -m "update website"`：保存一次版本
- `git push`：上传到 GitHub，并触发 Vercel 自动部署

## 8. 常见问题

### 页面本地正常，但线上没更新

可能是 Vercel 还没部署完成。

等待一会儿，再刷新线上页面。

也可以使用浏览器无痕窗口查看。

### git push 失败

先确认网络能访问 GitHub。

如果浏览器能访问 GitHub，但 Git 不能推送，可能是代理问题。

### npm.cmd run build 失败

说明代码或资源路径有问题。

先不要推送，修复后再重新运行：

```bash
npm.cmd run build
```

### 修改图片或视频后需要重新部署吗

需要。

只要你希望线上网站也更新，就需要：

```bash
git add .
git commit -m "update media"
git push
```
